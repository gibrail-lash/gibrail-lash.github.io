Process
Interpret

I believe the most critical phase in the design process is to define - and comprehend - the problem. After all, what use is a solution if it doesnt actually solve the problem. To fully understand a problem, I delve into questions, related and unrelated, that may need to be asked during any stage of the project. Steps in this process include identifying user groups, design constraints, and functional requirements.

Ideate

Once a problem is fully defined, I proceed to formulate possible solutions. Systems design is considered; specifying product architecture, which is a detailed description of each level of system hierarchy. Once all systems are developed, we can proceed to the final phase.

Implement

The final phase includes creating design concepts that will go through a structured comparison process based on product and user requirements. These include considerations such as safety, affordability, durability, etc. Once the design concepts have been quantitatively ranked, the 'best' designs are identified and a refinement process is engaged until a final design is considered and is used to create a detailed design.
